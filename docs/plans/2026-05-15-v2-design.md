# Roblox Leaderboard v2 ‚Äî Design

**Date:** 2026-05-15

## Overview

v2 builds on the existing Flask + vanilla JS app with three improvements: richer data, better visuals, and a more live feel. v1 stays untouched on `main`; v2 lives on a `v2` branch.

## Features

### More Data & Features

- **Player count history** ‚Äî server keeps the last 12 refreshes (‚âà1 hour) per game in memory
- **Sparklines** ‚Äî each leaderboard row shows a mini SVG line graph of the last hour's player count
- **Rank change column** ‚Äî shows spots moved since the previous refresh (e.g. ‚Üë3, ‚Üì1, ‚Äî)
- **Search bar** ‚Äî filters the leaderboard by game name in real time (client-side, no server call)

### Better Visuals

- **Animated counters** ‚Äî player counts count up/down when the leaderboard refreshes
- **Row highlight flash** ‚Äî rows that changed rank flash green (climbed) or red (dropped) briefly
- **Fade transition** ‚Äî smooth opacity fade when data reloads instead of instant swap

### Live Feel

- **2-minute refresh** ‚Äî interval drops from 5 minutes to 2 minutes (server and client)
- **Pulsing trend indicators** ‚Äî ‚Üë/‚Üì arrows pulse with a CSS animation
- **Big mover banner** ‚Äî a banner appears at the top when any game jumps or drops 5+ spots since the last refresh

## Architecture Changes

### Backend (`server.py` / `roblox.py`)

- `REFRESH_SECONDS` drops from 300 to 120
- Cache gains a `history` dict: `{ universeId: [playerCount, ...] }` ‚Äî last 12 values per game
- Each game object gains `rankChange` (int, positive = climbed) computed from previous snapshot
- History and rankChange are included in `/api/games` response

### Frontend (`public/`)

- `REFRESH_MS` drops from 300000 to 120000
- `renderLeaderboard` gains sparkline SVG generation per row
- `renderLeaderboard` gains rank change column
- Animated counter: on refresh, numbers tween from old value to new over 600ms
- Row flash: CSS class `flash-up` / `flash-down` added on refresh, removed after animation
- Big mover banner: rendered above the table if any game moved ‚â•5 spots
- Search bar: input above the table, filters rows client-side on `input` event
